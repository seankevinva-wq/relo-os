Phase 1 — The Three Modules

Each module is an independent automation. They share Emma as the AI persona and the RSG admin portal as the data source.

MODULE 01
Emma Warm Handoff
When a new order is placed, Emma texts the homeowner automatically — confirms details, collects missing info (make/model, images, gate codes), marks job confirmed. Replaces ~700 manual CS calls/month.
MODULE 02
After-Hours AI Phone
Emma answers the RSG phone after hours (8 PM – 7 AM). Recognizes caller ID, pulls the active job, answers common questions, collects missing info. Nick gets a text if escalation needed.
MODULE 05
Service Window Collection
Every morning Emma texts all vendors with upcoming jobs — "Select your window for [job] on [date]." Vendor replies with a time. Window auto-populates into the system. Eliminates Nick's daily call routine.

How the Pieces Fit Together

All three modules read from and write back to RSG's admin portal via Bitopan's API. Emma is the outbound channel (SMS/phone). RSG's portal stays the source of truth.

Questions for Bitopan

Is there a webhook system, or will we need to poll?
Webhooks = Emma reacts instantly to new orders. Polling = we hit the API every minute. Either works but webhook is cleaner.
Is homeowner phone number a required field on every order?
If it's optional, we need a fallback — Emma can't text without a number. Need to know the % of orders where it's missing.
Do vendor profiles include a mobile phone number?
Module 05 sends morning SMS to vendors. If email only, we need to discuss adding SMS numbers to profiles.
Is there a structured "info checklist" per job, or is it free-form notes?
Emma needs to know field-by-field what's missing. If it's already structured in the portal, we read it directly. If not, we build the logic ourselves.
Can we write inbound SMS replies back to a job record?
When a homeowner replies to Emma's text, that data needs to land on the job. If the portal supports it, great. If not, we store it on our side and sync.
Is there a staging/sandbox environment for testing?
We need to run tests without touching live customer data. If there's no sandbox, can Bitopan create a test account?

Questions for Nick

What time do you want Emma to send the morning blast?
Vendors are tradespeople — too early is annoying, too late leaves no time to sort problems. 7–8 AM likely sweet spot.
What are the standard window options you offer? (e.g. 8–11 / 11–2 / 2–5)
We need to know whether Emma should present fixed options ("Reply 1, 2, or 3") or accept free-text. Fixed options are far more reliable for parsing.
How far out should jobs be included in the blast? Next day only, or next 3 days?
If a job is a week out, do vendors need a window set now? Or is that too early? Nick knows the right threshold.
If a vendor doesn't reply to Emma's blast, how long before re-nudge? And before escalation to Nick?
e.g. 2 hours → second text. 4 hours → Nick gets alerted. Needs to be defined.
Once a window is confirmed, who gets notified — homeowner only, or also the moving company client (e.g. Jade at Syracuse)?
From the discovery call Jimmy mentioned wanting the notification to go to the homeowner automatically. But the moving company (Jade) was also mentioned as wanting visibility. Confirm scope for Phase 1.
